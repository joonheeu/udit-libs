# 보안 정책

## 지원되는 버전

현재 지원되는 버전의 보안 업데이트를 제공합니다.

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## 취약점 보고

보안 취약점을 발견한 경우, GitHub Issues를 통해 공개적으로 보고하지 마세요. 대신 다음 이메일로 직접 연락해주세요:

**security@udit.dev** (예시 - 실제 이메일로 변경 필요)

보고 시 다음 정보를 포함해주세요:

- 취약점의 유형 및 심각도
- 취약점을 재현하는 단계
- 가능한 경우 수정 제안
- 영향받는 패키지 및 버전

보고 후 48시간 이내에 응답을 드리며, 취약점을 확인한 후 수정 작업을 진행합니다.

## 보안 업데이트

보안 취약점이 수정되면:

1. 패치 버전이 릴리스됩니다
2. CHANGELOG에 보안 수정 사항이 기록됩니다
3. 영향을 받는 사용자에게 알림이 전송됩니다 (심각한 경우)

## 보상 프로그램

현재 보상 프로그램은 운영하지 않지만, 기여에 감사드립니다.
